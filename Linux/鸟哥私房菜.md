## 2.2、磁盘分区

### 2.2.1、磁盘连接方式与设备文件名的关系

正常的物理机器都使用/dev/sd[a-p]的文件名

### 2.2.2、MBR(MS-DOS)与GTP磁盘分区表

* 主引导记录 MBR(MS-DOS)：可以安装启动引导程序的地方，有466个字节
* 分区表 GTP：记录整块磁盘分区的状态，有64字节（最多只有4组记录区）
* 分区的目的：数据的安全性、系统的性能

### 2.2.3、启动流程中的BIOS与UEFI启动检测程序

* BIOS 搭配 MBR/GPT 的启动流程

> BIOS 是计算机系统启动主动执行的第一个程序

    1.BIOS:启动程序的固件，会认识第一个可启动的设备
    2.MBR :第一个可启动设备的第一个扇区内的主引导记录块，内涵启动引导代码
    3.启动引导程序：一个可读区内核文件来执行的软件
    4.内核文件：开始启动操作系统

* UEFI BIOS 搭配 GPT 启动的流程

> GPT 提供64位的寻址，但是 BIOS 不懂 GTP ，所以需要 GTP 提供兼容模式才能读写这个磁盘。因此 UEFI 统一扩展接口就产生了。

|比较项目|传统 BIOS| UEFI |
|--|--|--|
|使用程序语言|汇编语言|C语言|
|硬件资源控制|使用中断（IRQ）管理<br>不可变的内存读取<br>不可变输入输出读取|使用驱动程序与协议|
|处理运行环境|16位| CPU 保护模式|
|扩充方式|通过IRQ连接|直接加载驱动程序|
|第三方厂商支持|较差|较佳且可支持多平台|
|图形能力|较差|较佳|
|内置简化操作系统环境|不支持|支持|

## 5.2、Linux文件权限概念

### 5.2.1、Linux文件属性

    drwxr-xr-x      2        wfl        wfl          4.0K       1月  28 02:23     Public
    [文件类型权限][连接数] [文件拥有者] [文件所属用户组] [文件大小] [文件最后被修改时间] [文件名]

* 第一个字符代表这个目录、文件或连接文件等等
* 当为【d】则是目录
* 当为【-】则是文件
* 若是【|】则表示链接文件
* 若是【b】表示设备文件里面可供存储的周边设备
* 若是【c】则是设备文件里的串行端口设备
* 接下来的字母以3个为一组，r可读，w可写，x可执行，-无权限
* 第一组：文件拥有者可具备的权限
* 第二组：加入用户组之后的权限
* 第三组：非本人并且非本用户组的其他用户权限
* Linux文件的重要性
  
1. 系统保护的功能
2. 团队开发的软件或数据共享的功能
3. 未将权限设置妥当的危害

### 5.2.2、如何修改文件的属性权限

* chowm：修改文件拥有者

        比如将root权限的文件修改成普通的用户权限

* chmod：修改文件权限

        数字类型权限修改文件权限
            给rxw进行赋值r：4，w：2，x：1

            +（加入）、-（移除）、=（设置）

## 5.3、Linux目录配置

### 5.3.1、Linux目录配置的依据———FHS

| |可分享|不可分享|
|--|--|--|
|不变|/usr(软件存放处)|/etc(配置文件)|
||/opt(第三方辅助软件)|/boot(启动与内核文件)|
|可变动|/var/mail(用户邮箱)|/var/run(程序相关)|
||/var/spool/news(新闻组)|/var/lock(程序相关)|

> FHS 建议根目录所在分区越小越好，应用与所安装的软件最好不要与根目录放在同一个分区内，保持根目录越小越好

|目录|应放置文件内容|
|--|--|
|第一部分：FHS要求必须要存在的目录|
|/bin|bin放置单人维护模式下还能被使用的命令，bin命令可以被root与一般用户所使用，如：cat、chmod、chown、data等等|
|/boot|放置启动使用到的文件，Linux内核常用文件名为：vmlinuz，grub也在此目录下|
|/dev|任何的设备接口都是以文件的形式存在于这个目录中，只要读取目录下的某个文件就是读取某个设备|
|/etc|系统的只要配置文件基本上都放在/etc中，FHS建议不要将可执行文件放在这个目录中|
|/lib|放置启动时会用到的函数库，以及在/bin或/sbin下面命令的会调用的函数库|
|/media|放置可删除的文件|
|/mnt|暂时挂载某些额外设备|
|/opt|第三方辅助软件放置目录|
|/run|用来模拟内存，性能会更好|
|/sbin|放在/sbin下面的是启动过程所需要的，里面包括启动、修复、还原系统所需要的命令|
|/srv|（service）是一些网络服务启动之后这些服务所需要使用的数据目录|
|/tmp|正在执行的程序暂时存放文件的地方，这个文件任何人都可以存取，需要定期清理|
|/usr|第二层FHS设置|
|/var|第二层FHS设置，只要放置变动性数据|
|第二部分：FHS建议可以存在的目录|
|/home|系统默认用户家的目录|
|/lib< qual >|用来放置不同格式的二进制函数库，例如支持64位的/lib64函数库等等|
|/root|系统管理员（root）的家目录|
|/lost+found|使用标准的ext2,ext3,ext4才会产生的一个目录，目的在于当系统文件丢失时将一些遗失的片段放在目录下|
|/proc|这个文件本身是虚拟文件系统，他放置的数据都在内存之中，本身并不占用外存空间|
|/sys|与/proc相似，本身也是一个虚拟文件系统。本身记录内核与系统硬件的相关信息|

|目录|应放置文件内容|
|--|--|
|第一部分：FHS要求必须要存在的目录|
|/usr/bin|一般用户使用的命令，也就是说/bin与/usr/bin是一模一样的|
|/usr/lib|与/lib功能相似，所以/lib就是链接到此目录中|
|/usr/local|系统管理员安装下载的软件（非发行版默认提供的软件）|
|/usr/sbin|非系统正常运行所需要的系统命令，最常见的是网络服务器软件所需要的命令|
|/usr/share|只要放置只读文件还有共享文件，几乎都是文本文件<br>/usr/share/man在线帮助文档<br>/usr/share/doc软件说明文档<br>/usr/share/zoneinfo与时区有关的时区文件|
|第二部分：FHS建议可以存在的目录|
|/usr/games|与游戏相关的文件|
|/usr/include|c/c++的程序语言的头文件与包含文件放置处|
|/usr/libexec|不被一般用户常用的执行文件或脚本|
|/usr/lib< qual >|与/lib< qual >功能相同，/lib< qual >链接到此目录中|
|/usr/src|一般源代码放到这里|

|目录|应放置文件内容|
|--|--|
|第一部分：FHS要求必须要存在的目录|
|/var/cache|应用程序本身运行过程中产生的一些缓存|
|/var/lib|程序本身执行过程中，需要使用的数据放置的目录。此目录下每个软件都有自己的目录|
|/var/lock|某些文件或设备资源只能被一个应用程序所使用，若有两个程序使用该设备就肯可能产生错误，因此要将该设备上锁，以确保该设备只有一个软件所使用|
|/var/log|日志文件所放置的目录比较重要的有/var/log/messages、/var/log/wtmp等等|
|/var/mail|放置个人邮箱的目录，不过也被放到/var/spool/mail，这两个地方为相互链接的文件|
|/var/run|某些程序或服务启动后，会将他们的PID放置在这个目录下,与/run想来链接|
|/var/spool|一般放置一些队列数据,所谓队列数据就是排等待其他程序使用的数据,这些数据用完之后会被删除|

## 6.2、目录与文件管理

### 6.2.1、文件与目录的查看:ls

> ls [-aAdfFhilnrRSt] 文件名或目录名称  
> ls [--color={never,auto,always}]  
> ls [--full-time] 文件名或目录名称
>> <u>-a 全部文件,连同隐藏文件(常用)</u>  
>> -A 全部文件,连同隐藏文件,但不包括(.与..)这两个目录  
>> <u>-d 仅列出目录本身(常用)</u>  
>> -f 直接列出结果,不对文件进行排序  
>> -F 根据文件,目录等给予附加数据结构  
>> -h 将文件容量列出来  
>> -i 列出inode号码  
>> <u>-l 详细信息显示,包含文件的属性与权限(常用)</u>  
>> -n 列出UID与GID而非使用者与用户组的名称
>> -r 将排序结果反向输出
>> -R 连同子目录一起列出
>> -S 以文件容量大小排序
>> -t 依时间排序
>> --color=never 不要依据文件特征给予颜色
>> --color=always 显示颜色
>> --color=auto 系统依据设置自行判断是否给予颜色
>> --time=[atime,ctime] 输出access时间或改变权限属性时间(ctime)

### 6.2.2、复制、删除与移动:cp、rm、mv

* cp 复制文件或目录

> cp [-adfilprsu] 源文件 目标文件
>> <u>-a 相当于-dr --preserve=all的意思(常用)</u>  
>> -d 若源文件为链接文件,则复制链接文件属性而非文件本身  
>> -f 为强制的意思若目标文件已经存在且无法开启,则删除后再尝试一次  
>> <u>-i 若目标文件已经存在,在覆盖是会先询问操作的进行(常用)</u>  
>> -l 进行硬链接的链接文件,而非复制文件本身  
>> <u>-p 连同文件属性(权限,用户,时间)一起复制过去,而非使用默认属性(备份常用)</u>  
>> <u>-r 递归复制,用于目录的复制操作(常用)</u>  
>> -s 复制成为符号链接文件,亦称快捷方式  
>> -u destination比source旧才更新,或destination不存在的情况下才复制  
>> --preserve=all 除了-p的权限相关参数外,还加入SELinux的属性,links xattr等也复制

* rm 删除文件或目录

> rm [-fir] 文件或目录
>> -f 忽略不存在的文件,不会发出警告
>> -i 交互模式,删除前询问使用者是否操作
>> -r 递归删除,常用于删除目录,<u>这个选项比较危险</u>

* mv移动文件或目录,或重命名

> mv [-fiu] source destination
> mv [options] source1 source2 ... directory
>> -f 若文件已经存在,不会询问直接覆盖
>> -i 若目标文件已经存在时询问是否存在
>> -u 若目标已经存在且比source新,才会更新

### 6.3.1、直接查看文件内容

* cat

> cat [-AbEnTv]
>> -A 相当于-vET的整合,可列出一些特殊的字符而不是空白而已  
>> -b 列出行号,仅针对非空白行做出的行号显示,空白行不标行号  
>> -E 降结尾换行以$显示出来  
>> <u>-n 打印出行号,空白行也有行号,与-b的选项不同</U>  
>> -T 将tab键以^I显示出来  
>> -v 列出一些看不出来的特殊符号  

* nl 添加行号打印

> nl [-bnw] 文件  
>> -b 指定行号指定的方式,有两种  
>>> -b a 表示无论是否为空行同样列出行号  
>>> -b c 如果有空行,空的那一行不列行号  
>>  
>> -n 列出行号的表示方法,有三种
>>> -n ln 行号在屏幕最左方显示  
>>> -n rn 行号在自己栏位的最右方显示,不加0  
>>> -n rz 行号在自己栏位的最右方显示,加0  
>>  
>> -w 行号栏的占用的字符数

### 6.3.2、可翻页查看

* more 文件

        空格键:代表向下翻一页

        /字符串:代表在这个显示的内容中向下查找字符串

        空格键:代表向下翻一页
        [pagedown]:代表向下翻一页

        /字符串:代表向下查找字符串的功能

        n   :重复前一个查找(与/和?有关)
        N   :反向的重复前一个查找(与/和?有关)
        g   :前进到这个数据的第一行
        G   :前进到这个数据的最后一行
        q   :离开less这个程序
